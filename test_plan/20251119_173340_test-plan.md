# Application - Feature Test Plan
## Application Overview
This test plan outlines the steps to delete an existing account in Salesforce's Accounts module. The process involves navigating to the Accounts page, selecting an account, and confirming its deletion.

## Test Scenarios
### 1. Delete an Existing Account
**Seed:** tests/seed.spec.ts

#### 1.1 Delete Test Account 3
**Steps:**
1. Navigate to the Salesforce home page.
2. Click on the "Accounts" link in the navigation menu.
3. Click on the "Show Actions" button for "Test Account 3".
4. Select the "Delete" option from the dropdown menu.
5. Confirm the deletion by clicking the "Delete" button in the confirmation dialog.

**Expected Results:**
- Step 1: The Salesforce home page should load successfully.
- Step 2: The Accounts page should display a list of accounts, including "Test Account 3".
- Step 3: The actions menu for "Test Account 3" should open, displaying the "Delete" option.
- Step 4: The account should be marked for deletion.
- Step 5: A success notification should appear confirming that "Test Account 3" was deleted.

### Edge Cases
- Attempt to delete an account that does not exist should show an error message.
- Confirming deletion without selecting an account should not proceed with any action.
