# Application - Feature Test Plan
## Application Overview
This test plan outlines the steps to create a new account in Salesforce within the Accounts module. The process involves navigating to the Accounts section, filling in the required fields, and saving the new account.

## Test Scenarios
### 1. Create Account in Salesforce
**Seed:** tests/seed.spec.ts

#### 1.1 Create Account with Minimum Required Data
**Steps:**
1. Navigate to the Accounts module.
   - Action: Click on the "Accounts" link.
   - Element: `getByRole('link', { name: 'Accounts', exact: true })`
2. Click on the "New" button to create a new account.
   - Action: Click on the "New" button.
   - Element: `getByRole('button', { name: 'New', exact: true })`
3. Fill in the "Account Name" field.
   - Action: Type "Test Account 3" into the "Account Name" textbox.
   - Element: `getByRole('textbox', { name: 'Account Name', exact: true })`
   - Expected Result: The textbox should display "Test Account 3".
4. Click on the "Save" button to save the new account.
   - Action: Click on the "Save" button.
   - Element: `getByRole('button', { name: 'Save', exact: true })`
   - Expected Result: The URL should change to `/lightning/r/Account/[id]/view`, indicating the account has been created successfully.

**Expected Results:**
- Step 1: The user should be navigated to the Accounts page.
- Step 2: The New Account form should be displayed.
- Step 3: The "Account Name" field should be populated with "Test Account 3".
- Step 4: The account should be saved, and the user should be redirected to the account view page.

### Edge Cases
- Attempt to save without filling in the "Account Name" field.
  - Expected Result: An error message should be displayed indicating that the "Account Name" is required.
- Fill in the "Account Name" with special characters.
  - Expected Result: The account should be saved successfully if the system allows special characters.
